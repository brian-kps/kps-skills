# Linear Workflow

Follow the team's Linear conventions when creating, triaging, or organizing issues.

## Quick Reference

- **Theme projects** = product area backlog (Event Experience, Payments, etc.)
- **Version projects** = release execution plan (v1.0.0 Release, etc.)
- **Issues graduate** from theme project -> version project when committed to a release
- **Type labels** (single-select): Feature, Bug, Improvement, Chore
- **Platform labels** (single-select): iOS, Android, Web, Server, Shared — only apply when one platform dominates
- **No cycles** — we use version-based releases with milestones, not sprints
